## Project Checkpoint Summary

You are an expert project manager and technical writer.

Create a **concise but complete "Project Checkpoint Summary"** that I can easily copy-paste into a brand new conversation.

Include exactly these sections using clear markdown formatting:

### 1. Project Goal
- The original main objective of this project

### 2. Key Decisions Made So Far
- Major choices and directions we have taken
- Important design, technical, or structural decisions

### 3. Current State
- What has been completed so far
- Main files, sections, or components already worked on
- Any important outputs, code structures, or results generated

### 4. Next Steps
- What we were planning to do next (be specific)

### 5. Important Artifacts
- Key code snippets, diagrams, structures, or other important outputs worth keeping (keep them short and relevant)

**Rules:**
- Keep the entire summary under 4000 words (ideally 800–2000 words max)
- Focus only on the most important and useful information
- Be clear, structured, and easy to copy-paste
- Use bullet points and subheadings for readability
- Do not add extra explanations or filler text outside the defined sections

Now generate the checkpoint summary based on everything we have done in this conversation so far.
